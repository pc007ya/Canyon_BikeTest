GitHub 部署教學

把測試制具查詢系統放到 GitHub，用 GitHub Pages 免費架設網址。全程約 10 分鐘，不需要任何伺服器或建置流程。

純靜態網站 免費 · 無需信用卡 Firebase 雲端同步照常運作
i
這個系統是純前端（HTML + JS，瀏覽器內即時轉譯），沒有建置步驟，GitHub Pages 可以直接原封不動架起來。雲端同步（Firebase）是純前端呼叫，跟主機在哪裡無關，搬過去後行為完全一樣。
1

建立 GitHub Repository

github.com

登入 github.com → 右上角 + → New repository。

Create a new repository
Repository nametest-fixture-finder
Visibility● Public　○ Private

名稱隨意；選 Public（Private 也可以用 Pages，但需付費方案）。不要勾選「Add a README」，保持空倉庫即可。

2

上傳專案檔案

兩種方式擇一

方式 A · 網頁拖曳上傳（不需裝 Git，最簡單）

  1. 進入剛建立的空倉庫頁面，點 uploading an existing file。
  2. 把下載的部署資料夾「整個資料夾內容」（不含最外層資料夾本身）拖進瀏覽器視窗——index.html 必須直接在倉庫根目錄，不能藏在子資料夾裡。
  3. 下方填寫 commit 訊息（例如「Initial deploy」），按 Commit changes。
!GitHub 網頁上傳一次最多約 100 個檔案／單檔 25MB。這個專案檔案數在範圍內，若之後 uploads/ 圖片變多超過限制，改用方式 B。

方式 B · Git 指令（適合之後要常常更新）

Terminal···
$ cd /path/to/專案資料夾 $ git init $ git add . $ git commit -m "Initial deploy" $ git branch -M main $ git remote add origin https://github.com/你的帳號/test-fixture-finder.git $ git push -u origin main

「你的帳號」與倉庫名稱請換成你自己的；GitHub 會在建倉庫頁面直接給你這串指令，複製貼上即可。

4

之後要更新內容怎麼辦？

後續維護

方式 A（網頁上傳的人）：回到倉庫，進入要改的檔案 → 點鉛筆圖示編輯，或用「Add file → Upload files」覆蓋整批，commit 後 Pages 會在 1 分鐘內自動重新部署，不用重新設定。

方式 B（Git 指令的人）：改完檔案後執行：

Terminal···
$ git add . $ git commit -m "Update" $ git push

GitHub Pages 每次 push 到 main 都會自動重新發佈，通常 30 秒內生效。

✓

需要放哪些檔案？

部署資料夾內容

把下載的部署資料夾「內容」整個放進倉庫根目錄（index.html 要在最上層）：

index.htmlstyles.css app.jsxadmin.jsx admin-manage.jsxcomponents.jsx fixtures.jsxequipment.jsx equipment.jsnotify.jsx reports.jsxhistory.jsx tweaks-panel.jsxdeck-stage.js store.jscloud.js auth.jstheme.js idb.jszip.js data.jsjigs.js tests.jsjigs/ (資料夾) uploads/ (資料夾)供應商流程指引.html
!_redirects 這個檔案是 Netlify 專用設定，GitHub Pages 不認得它，留著也沒關係（會被忽略），不留也不影響任何功能——這系統用的是 #/... 網址片段做路由，本來就不需要伺服器轉址規則。
★

雲端同步（Firebase）需要改設定嗎？

不需要

index.html 裡已經寫死你的 Firebase 專案設定，純前端呼叫，跟部署在哪個網域無關，搬到 GitHub Pages 後雲端同步照常運作，不用重新設定 Firebase。

i唯一要注意：Firebase Authentication 的「已授權網域」清單，預設只允許 localhost 與你原本設定的網域。若管理員要在新網址上用「Google 登入」同步，需到 Firebase 主控台 → Authentication → Settings → Authorized domains，新增 你的帳號.github.io。
★

Tweaks 設計面板還能用嗎？

已修正，可正常使用

右上角語言切換旁新增了一顆 齒輪圖示，點下去即可開關「設計調整」面板（切換主題、密度、示意圖顯示），不依賴任何外部平台，GitHub Pages / Netlify / 本機開啟都能正常使用。

?

常見問題

開啟網址是 404
確認 index.html 是在倉庫「根目錄」，不是在子資料夾裡；Settings → Pages 的 Branch 是否已選對分支並按 Save。
畫面空白、主控台有紅字
多半是某個 .jsx/.js 檔沒上傳完整。對照上方「需要放哪些檔案」清單逐一核對。
圖片顯示不出來
jigs/ 與 uploads/ 兩個資料夾要整包上傳，注意檔名大小寫需與程式碼內參照完全一致（GitHub Pages 對大小寫敏感，Windows 本機測試可能不會，上線後才發現）。
Google 登入同步失敗
依上方「Firebase 需要改設定嗎」段落，把新網域加入 Firebase Authentication 的已授權網域清單。
